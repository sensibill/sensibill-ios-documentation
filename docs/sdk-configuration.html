SDK Configuration

When the SDK starts up it will attempt to find the Sensibill.plist configuration file in the main bundle of the host target app.

  1. Create Sensibill.plist in the main bundle of your target.
  2. Specify values, depending on the authentication type you are planning to use, as explained in Configuration Parameters the following section.
  3. Additionally, regardless of authentication type, add the Certificate Pinning section, as explained in corresponding section below.

Configuration Parameters

If using a PasswordAuthenticator, specify the following values:

API_KEY: String
API_SECRET: String
CREDENTIAL_TYPE: String
REDIRECT_URL: String

If using an Integration Server and custom TokenProvider, specify an environment URL in the following parameter:

ENVIRONMENT: String

Note: Please ensure the URL provided here does not have a trailing slash.

Certificate Pinning

Create the following key in the Sensibill.plist file:

CERTIFICATE_PINNING_FILES: Array of Strings

If certificate pinning on the SDK is enabled (see further information below), the array must contain a list of valid certificate names. The array can be empty if certificate pinning is disabled.

By default, certificate pinning on the SDK is enabled. For certificate pinning to function, the host app’s Target must include the certificates provided by Sensibill (COMODO.cer and GLOBOSIGN.cer), and must add the names of the certificates to the Sensibill.plist configuration file:

  1. Import certificates provided by Sensibill into your project
  2. Include them as a Bundle resources in every app’s target that uses Sensibill SDK.

  3. Add the names of the certificate files to the CERTIFICATE_PINNING_FILES parameter of Sensibill.plist:

       CERTIFICATE_PINNING_FILES
       Item 0: COMODO.cer
       Item 1: GLOBALSIGN.cer
