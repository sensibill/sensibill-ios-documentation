Customization

Branding Customization

There are two layers to branding customization. The easiest way for most purposes is to use the colors property of the SBSDKConfiguration singleton object. The properties of colors will be used by the SDK to map your branding to its visual elements. When the SBSDKConfiguration singleton is first accessed, it will be initialized with a default set of color values, but you can override these before starting the SDK:

let configuration = SBSDKConfiguration.sharedInstance()
configuration?.colors.brandBackground = . red
configuration?.colors.brandForeground = . blue
configuration?.colors.actionBackground = .gray
configuration?.colors.actionForeground = .white
configuration?.colors.accentBackground = .black
configuration?.colors.accentForeground = .white
configuration?.colors.alternateBackground = .green
configuration?.colors.alternateForeground = .yellow
configuration?.colors.errorBackground = .red
configuration?.colors.primaryBackground = .blue
configuration?.colors.offPrimaryBackground = .black
configuration?.colors.primaryForeground = .white
configuration?.colors.primaryForeground2 = .green
configuration?.colors.primaryForeground3 = .blue
configuration?.colors.primaryForeground4 = .red
configuration.applyColors()

There are plenty of other colors one can set, but these are just some examples. Using the small number of properties on the color object, you can quickly brand the SDK with minimal work.

Since you are setting these properties on the singleton SBSDKConfiguration.sharedInstance() object, there are no other changes needed (i.e. you don’t need to pass this object anywhere) as any UI created in the SDK will use these new values. All you need to do is call applyColors when you are done setting your custom colors on the SBSDKConfiguration singleton and these colors will be mapped to their respective elements.

The second way of setting these values (which could be used together with the first method) is to override the underlying properties which those initial properties map to. The only reason you would want to use this method was if you wanted an individual element on one screen to appear differently than a similar element on another screen. Usually these two elements would have the same appearance (they’d both use the child property of colors, as shown above), but in this case you may want to have more granularity:

let configuration = SBSDKConfiguration.sharedInstance()
configuration?.loadingViewConfiguration.loadingBackgroundColor = .black
configuration?.loadingViewConfiguration.loadingSpinnerColor = .white

Above you can see a simple example for overriding the colors of the loading view. If you didn’t override the colors like this, they would take on the colors defined in the higher-level customizations (in this example, brandBackground & brandForeground).

Font Customization

To customize the font in the Sensibill SDK, the SBSDKConfiguration instance provides a way to update the fonts in the SDK UI.

  1. Get the shared instance of the SBSDKConfiguration that the SDK reads from by calling SBSDKConfiguration.sharedInstance()

  2. Access the shared instance font configuration by calling SBSDKConfiguration.sharedInstance().font

  3. To change the font, adjust the properties of the font customization in the shared instance. To change the bold font, assign the desired bold font to SBSDKConfiguration.sharedInstance().font.boldFont. To change the regular font, assign the desired font to SBSDKConfiguration.sharedInstance().font.regularFont

Asset Customization

To customize assets in the Sensibill SDK, an external Bundle must be added to the SDK first. The SDK will use this added bundle to check for assets to use in the Sensibill UI. The assets inside of the Bundle must have the same name as assets already used in the SDK. By doing so, the provided assets override what is in the SDK.

  1. Follow Using External Bundle instructions to create an external bundle, and to provide its name to the SDK using SBSDKConfiguration.
  2. Inside the Bundle created in Step 1, create an Asset Catalog called SensibillAssets.xcassets

Image Customization

Once you have provided an external bundle that the Sensibill SDK will use to retrieve assets, and created an Asset Catalog, images in the SDK can be overridden.

  1. Follow the steps in Asset Customization first

  2. Prepare the desired imageset and rename it to match the imageset you want to replaced. For example, you can name an imageset to capture-close to override the close icon in the Receipt capture screen

  3. Copy the imageset from step 1 into the Asset Catalog SensibillAssets.xcassets from the Asset Customization steps

String Customization

Similar to Image customization, strings in the SDK can be customized by adding a Localizable.strings file to the external bundle.

  1. Follow the steps in Asset Customization first

  2. Prepare Localizable.strings with strings to override. The keys of your strings should match keys of strings in the SDK you would like to override. The value of your string should be the content you want to override with

  3. To support multiple languages, you can add multiple Localizable.strings language files to your external Bundle.
