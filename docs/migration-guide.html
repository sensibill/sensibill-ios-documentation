Upgrading from 1.26 to 2021.0.0

Summary of Changes

Integration must be updated to reflect the following changes:

  • The name of the framework changed from SensibillUI to Sensibill. See Required Updates section below.
  • The minimal supported iOS version was updated from iOS 11.0 to iOS 12.0. See Required Updates > Project Settings section below.

Required Updates

Installation

Via CocoaPods
  • Update the name of the module from SensibillUI to Sensibill in the Podfile.
Direct Download

  • In the Target settings, change Sensibill framework name from SensibillUI.framework to Sensibill.framework in the Build Phases section of the settings:

    • In Embed Frameworks phase
    • In Link Binary with Libraries phase
    • In Run Script Phase

Target’s Build Settings

  • Update your project’s Build Settings > iOS Deployment Target to iOS 12.0 or higher.

  • To use Sensibill module with Swift, update the module name from SensibillUI.framework to Sensibill.framework in the Swift Compiler - Search Paths of the Build Settings section of the settings.

Classes

  • Replace import SensibillUI with import Sensibill in all Swift classes
  • Replace @import SensibillUI; with @import Sensibill; in all Objective-C classes
  • Replace #import <SensibillUI/SensibillUI.h> with #import <Sensibill/Sensibill.h> in all Objective-C classes
  • Replace #import <SensibillUI/SensibillUI-Swift.h> with #import <Sensibill/Sensibill-Swift.h> in all Objective-C classes

Removed

  • exitAfterCapture property has been removed from CaptureFlowCoordinator as a result of removal of receipt processing native screen. By default, after receipt is captured then capture flow will be dismissed.
  • SDK no longer attempts to process incomplete receipt uploading transactions created in SDK 1.23 or lower versions and stored in SensibillTransactions.txt file. Use latest SDK 1.x version to process such receipt uploads, or resubmit the related receipts. Note: Uploading of the incomplete receipt uploading transactions created in SDK 1.24 and above is supported without a change.
