Troubleshooting

Installation Troubleshooting

Compilation error Could not load underlying module while importing SDK using import SensibillUI statement.

If you encounter this error while migrating from 1.25.0 to 1.25.2, please follow the How to use SensibillUI module in Swift instructions specified in the Direct Download section of the Installation Guide.

No such module ‘SensibillUI’ (in Swift File)

This compile time error will appear next to the following statement in a Swift file after upgrading to v1.25 if you are consuming the SDK by embedding it manually.

import SensibillUI
  • To fix this, add the path to the Sensibill framork to your target’s Import Paths setting under the Swift Compiler - Search Paths section in your target’s Build Settings. Specify the recursive option.

Module ‘SensibillUI’ not found (in Objective-C File)

This compile time error will appear next to following statement in Objective-C file after upgrading to v1.25.

@import SensibillUI;

Usage of this syntax is unavailable as of v1.25. Please convert to the following syntax:

#import <SensibillUI/SensibillUI.h>
#import <SensibillUI/SensibillUI-Swift.h>

Unknown type name SBL* (in Objective-C File)

This compile time error will appear in Objective-C file after upgrading to v1.25, even if you have imported SensibillUI with following syntax.

#import <SensibillUI/SensibillUI.h>

The following import statement is required to access the SDK’s swift classes in an Objective-C file:

#import <SensibillUI/SensibillUI-Swift.h>

Image not Found

You may get an image not found error during run time:

Referenced from: /.../SensibillDemo.app/Frameworks/SensibillUI.framework/SensibillUI
Reason: image not found

The following are common ways that this error is caused, and ways to mitigate them:

  • Framework was not included in Embedded Binaries
    • To fix this ensure that the framework is included in Embedded Binaries in addition to the default Linked Frameworks and Libraries.
  • Objective-C only project may not have Swift setup properly
    • To fix this simply add a single Swift file to your project using File > New > File…. When Xcode prompts you if you would like to create a Bridging Header click Yes.

No code signature Found

If you encounter an error when installing your app to a physical device:

App installation failed
No code signature found.

Make sure that the Run Script Phase added during installation has the option Run script only when installing checked.

Debugging Errors

All errors received or generated by the SDK are logged using os_log with the prefix [SBLError]. A SensibillError code will always be logged at the beginning of an error. This is an error code that will be returned to your app from the SDK. Example:

[SBLError] SensibillError=1001: Error Domain=<...>

Additionally, if an error is returned by the Sensibill API, it will contain a correlationID field, which allows Sensibill support to find a matching operation in server side logs.
