NetworkReachabilityService

@objc(SBLNetworkReachabilityService)
public class NetworkReachabilityService : NSObject

The NetworkReachabilityService class allows to determine the status of a system’s current network configuration, and lstens for status changes. It should not be used to prevent a user from initiating a network request, as it’s possible that an initial request may be required to establish reachability.

Recommended usage pattern:

  1. Subscribe to reachabilityDidChangeNotification early in lifecycle of a class. For example in init or formDidLoad.
  2. Right after subscription, check current value of either status or isReachable to establish initial state. No new notification will be sent until status change, hence it’s important to verify the state after subscription.
  3. Notification does not send out new status. Instead, once notification is received, subscriber code should check new status using either status or isReachable. This prevents a possibility that stale status passed to subscriber (in case it changes several time within short period of time)
