Authentication

The SDK supports the following authentication options.

  • Custom Token Provider Authentication is used to allow the host app to control token management. The SDK supports OAuth2-compatible tokens, and JWT tokens.
  • Password Authentication using Sensibill backend APIs and PasswordAuthenticator. Authenticate using a username and password, and allow the SDK to handle all token management.

Both options adhere to the OAuth 2.0 standard.

Custom Token Provider Authentication

With this option all authentication, authorization, and token management operations are performed on the host app, and the app provides tokens to the SDK through the provideTokenReplacement function of a TokenProvider instance.

Prerequisites

Your Sensibill.plist requires the environment URL string to be set for the key ENVIRONMENT. Additionally, you should set the CERTIFICATE_PINNING_FILES if certificate pinning is enabled. See Setup for more information.

Steps

  1. Create a custom token provider to handle creating and refreshing tokens. The custom token provider must adhere to the TokenProvider protocol.
  2. Pass the custom token provider instance to the SDK on start. See SDK Lifecycle for further information.
  3. The SDK will retain an instance of the custom token provider until it is stopped.

Custom Token Provider Example

import SensibillUI

class CustomTokenProvider: TokenProvider {
    func provideTokenReplacement(completion: @escaping (Credentials?, Error?) -> ()) {

        // Obtain a token from your integration server
        var request = URLRequest(...)
        let task = URLSession.shared.dataTask(with: request) { (data, response, error) in

            // On callback from your authentication server

            // 1. Check for error. If error is returned, pass this error in callback to SDK:
            if error != nil {
                completion(nil, error)
                return
            }

            // 2. Parse response to obtain access token
            let accessToken = ...

            // 3. Construct a Credentials object and return it in a callback to SDK:
            let credentials = Credentials.init(accessToken: accessToken)
            completion(credentials, nil)
        }
        task.resume()
    }
}

Password Authentication

This option allows the host app to take advantage of various authentication and user management operations through Sensibill’s backend API:

  • User registration flow (check username availability, password strength, register a user)
  • Login with username and password, and logout
  • Reset password

Prerequisites

Password authentication requires API_KEY, API_SECRET, CREDENTIAL_TYPE, and REDIRECT_URL to be specified in your Sensibill.plist. Additionally, you should set the CERTIFICATE_PINNING_FILES if certificate pinning is enabled. See Setup for more information.

A user also must be registered using PasswordAuthenticator‘s registration flow (see PasswordAuthenticator class documentation)

The login function requires a network connection, and cannot run in offline mode. For the logout function, having a network connection is optimal, but the user will be logged out locally in all cases.

Steps

To login

Perform these steps to start interacting with the SDK.

  1. Instantiate and retain an instance of PasswordAuthenticator.
  2. Call the login function to authenticate a user. This function will also start the SDK. See SDK Lifecycle for further information.
  3. login is an asynchronous operation. The caller must wait for the login callback to return and succeed before attempting any further interaction with SDK.

To logout

Perform these steps to stop interacting with the SDK for the current user. For example, when the user is about to log out from the app.

  1. Call PasswordAuthenticator’s logout to log out and stop SDK.
  2. logout is an asynchronous operation. The caller must wait for logout’s callback to return before attempting the next login operation.

Re-login

If necessary, you can call the login operation again (with the same or different credentials) once a logout operation is complete. However, login will fail if you attempt to call it 2 or more times without calling logout.

PasswordAuthenticator Example

import SensibillUI

let authenticator = PasswordAuthenticator()

//...

authenticator.login(accessId: "accessId", password: "password") { error in

    // Check for an error. If an error is returned, interaction with SDK will not be possible
    guard case .none = error else {
        // Add custom error handling
        return
    }

    // Start interacting with the SDK
}

//...

authenticator.logout()

Auto Login

For PasswordAuthenticator we provide a convenient feature to login as the last authenticated user, assuming the refresh token is still valid. To use this feature you can simply call the login method with a nil password. If this attempt fails, the user will be required to authenticate with their username and password.

authenticator.login(accessId: "accessId", password: nil) { error in

    // Check for an error. If an error is returned, the user must login with their username and password
    guard case .none = error else {
        // Add custom error handling. For example, navigate to your Login screen
        return
    }

    // Start interacting with the SDK
}
